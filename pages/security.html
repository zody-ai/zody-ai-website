Software security

How Zody AI Ltd validates software security through testing, review and regular maintenance of its technical controls.

Penetration testingPatch managementAudit review

Questions this page answers

Has the application been penetration tested?

The service is assessed through structured penetration testing and vulnerability scanning to identify and address weaknesses.

How are security controls reviewed?

Controls are reviewed periodically to confirm they remain aligned with current architecture and legal requirements.

How are vulnerabilities handled?

Dependencies and infrastructure are monitored so known vulnerabilities are identified and addressed promptly.

Security assurance approach

Zody AI Ltd treats software security as an ongoing discipline. The service is assessed through penetration testing and structured vulnerability scanning, with findings prioritised and remediated. Security controls are revisited as the architecture evolves, and dependency updates are monitored to reduce exposure to known vulnerabilities.

  • Penetration testing before major go-live milestones.
  • Periodic review of security controls against practical risk.
  • Patch and dependency management to address known issues.
  • Logging and monitoring that support anomaly detection and incident investigation.

Security evidence

  • Penetration testing carried out before major go-live milestones.
  • Dependency and vulnerability scanning maintained as part of the release process.
  • Security review findings tracked and remediated with documented outcomes.
