Software security

How Zody AI Ltd validates software security through testing, review and regular maintenance of its technical controls.

Penetration testingPatch managementAudit review

Questions this page answers

Has the application been penetration tested?

The service should be assessed with tools or an independent provider to identify exploitable weaknesses.

How are security controls reviewed?

Controls should be revisited periodically to confirm they still align with legal duties and current architecture.

How are vulnerabilities handled?

Dependencies, configuration and infrastructure should be monitored so known issues can be fixed promptly.

Security assurance approach

Zody AI Ltd treats software security as an ongoing discipline rather than a one-off exercise. The service should be assessed through penetration testing or equivalent structured testing, with findings prioritised and remediated. Security controls are revisited as the architecture evolves, and dependency updates are monitored to reduce exposure to known vulnerabilities.

  • Penetration testing before major go-live milestones.
  • Periodic review of security controls against practical risk.
  • Patch and dependency management to address known issues.
  • Logging and monitoring that support anomaly detection and incident investigation.

Useful evidence to retain

  • Recent penetration testing report summary.
  • Dependency or vulnerability scan outputs.
  • Security review notes and remediation tracking.
