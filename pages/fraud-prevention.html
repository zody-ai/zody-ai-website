Fraud prevention data

How software that uses VAT MTD or Income Tax Self Assessment MTD APIs captures and validates the fraud prevention headers HMRC requires.

Header dataValidationMTD APIs

Questions this page answers

Do you submit fraud prevention data where required?

Where HMRC APIs require it, we collect and transmit the necessary header data to identify the request environment and reduce fraud risk.

Have you validated the header implementation?

Headers are validated for correct format and end-to-end transmission before and after go-live.

How do you keep the data accurate?

The implementation minimises spoofing risk, documents the source of header values and is reviewed whenever the client or backend stack changes.

Implementation posture

Where Zody AI Ltd uses HMRC APIs that require fraud prevention data, the software collects and transmits the necessary header information in accordance with HMRC’s specification. That includes validating header format, confirming end-to-end behaviour before go-live and keeping the implementation under review as the stack evolves.

  • Fraud prevention headers only where the API requires them.
  • Controlled collection of device, application and request context.
  • Testing against HMRC’s fraud prevention validation path before go-live.
  • Review processes when mobile, web or proxy behaviour changes.

Applicable APIs

Fraud prevention header requirements apply to VAT MTD and Income Tax Self Assessment MTD APIs. Controls are enforced before any such integration goes live and reviewed whenever the implementation changes.
