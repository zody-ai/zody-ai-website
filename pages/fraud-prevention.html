07 · API integrity & fraud controls

API fraud prevention

How Zody AI Ltd implements request integrity controls, fraud prevention headers and validation for regulated API integrations including VAT MTD and Income Tax Self Assessment MTD.

Request integrityHeader validationRegulated APIs

Questions this page answers

Do you implement fraud prevention controls where required?

Where regulated APIs require fraud prevention data, we collect and transmit the necessary header values to identify the request environment and reduce fraud risk.

How is the implementation validated?

Headers are validated for correct format and end-to-end behaviour before go-live and after any significant change to the stack.

How is accuracy maintained over time?

The implementation documents the source of each header value, minimises spoofing risk and is reviewed whenever client or backend components change.

Implementation posture

Where regulated API integrations require fraud prevention data, Zody AI Ltd implements the necessary controls in accordance with the published specification. That means validating header format and content, confirming end-to-end behaviour before go-live and keeping the implementation under active review as the underlying stack evolves.

  • Fraud prevention headers submitted only where the API specification requires them.
  • Controlled, documented collection of device, application and request context.
  • Pre-go-live validation against the platform’s fraud prevention test tooling.
  • Review triggered whenever mobile, web or proxy components change.

Applicable integrations

Fraud prevention header requirements currently apply to VAT MTD and Income Tax Self Assessment MTD API integrations. Controls are enforced before any such integration goes live and reviewed whenever the implementation or stack changes.
